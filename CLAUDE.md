## Project Overview

Job Match Agent is a personal AI-powered job search tool that fetches listings from multiple job boards, scores them against a resume using Claude AI, generates tailored LaTeX resumes for top matches, and delivers a ranked HTML report to an inbox daily. Built as a personal tool during an active job search — not a SaaS product.

**Status:** Active development

---

## Stack

- **Runtime:** Node.js 18+ (ESM modules)
- **AI:** Anthropic Claude (`claude-sonnet-4-6`) via `@anthropic-ai/sdk`
- **Job Sources:** Adzuna API, The Muse API, Jobicy RSS
- **Email:** Nodemailer (Gmail + App Password)
- **Resume:** Plain text (`resume.txt`) + LaTeX (`resume.tex`) compiled via `pdflatex`
- **Dashboard:** Static HTML generated to `index.html`
- **Package manager:** npm

---

## Deployment

This is a local CLI tool, not a deployed service.

- **Run manually:** `node main.js`
- **Automate:** cron job — `0 7 * * * cd /path/to/job-agent && node main.js`
- **Dashboard:** Hosted via GitHub Pages at the repo's `main` branch (`index.html`)
- **Environment:** Copy `.env.example` → `.env`. Never commit `.env`.

## Project Structure

```
job-agent/
├── main.js                    # Master orchestrator (6-step pipeline)
├── config.js                  # Target cities, job titles, scoring thresholds
├── resume.txt                 # Plain text resume (used for AI parsing)
├── resume.tex                 # Master LaTeX resume (used for tailoring)
├── index.html                 # Auto-generated dashboard (committed for GitHub Pages)
├── src/
│   ├── parseResume.js         # AI resume parsing → skills/profile object
│   ├── jobSearch.js           # Multi-source job fetching + deduplication
│   ├── scoreJobs.js           # AI scoring (0–100%) + salary estimation
│   ├── database.js            # Persistent run history (30-day window)
│   ├── dashboard.js           # HTML dashboard generation
│   ├── emailSummary.js        # Daily email report with PDF attachments
│   └── tailorResumes.js       # Tailored LaTeX resume generator + pdflatex compile
├── data/
│   └── jobs.json              # Auto-generated run history (committed)
└── output/                    # Tailored .tex and .pdf files per run
```

## Conventions

- **Language:** JavaScript ESM (`type: "module"` in package.json) — always use `import`/`export`, never `require`
- **Naming:** camelCase for variables and functions, PascalCase for classes
- **Commits:** Conventional commits (`feat:`, `fix:`, `chore:`, `docs:`)
- **Error handling:** Never swallow errors silently. Always log with context.
- **Comments:** Explain *why*, not *what*. Remove debug comments before commit.
- **Config changes:** Target cities, job titles, and thresholds all live in `config.js` — don't hardcode them elsewhere.

---

## Do Not Touch

- `.env` — never read, never modify, never commit
- `data/jobs.json` — auto-generated; only modify via `database.js`
- `index.html` — auto-generated by `dashboard.js`; edit the generator, not the output
- `output/` — generated artifacts (tailored `.tex` and `.pdf` files)
- `resume.txt` / `resume.tex` — personal data, gitignored

## Known Issues

- `EXCLUDED_KEYWORDS` in `config.js` filters by title string match only — doesn't catch all seniority variations
- `pdflatex` must be on PATH or set via `TEX_BIN` env var; silently skips PDF compile if missing
